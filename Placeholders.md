## Placeholder Naming Conventions

### Format Structure:
```
{{[LEVEL]_[SECTION]_[TYPE]_[MODIFIER]}}
```

### Examples:
- `{{H1_SERVICE_PAGE_TITLE}}` - H1 heading for service page title
- `{{SERVICE_HERO_CTA_PRIMARY}}` - Primary CTA in service hero section
- `{{BLOG_POST_FEATURED_IMAGE}}` - Featured image for blog post
- `{{FOOTER_MENU_COMPANY_HEADING}}` - Company section heading in footer

### Guidelines:
1. **Use ALL CAPS** for placeholder names
2. **Use underscores** to separate words
3. **Be descriptive** - placeholder name should explain its purpose
4. **Include hierarchy** - H1, H2, H3 for headings
5. **Include location** - indicate which section/block it belongs to
6. **Include type** - TEXT, LINK, IMAGE, URL, etc.
7. **Use modifiers** - PRIMARY, SECONDARY, MAIN, ALT for variations
